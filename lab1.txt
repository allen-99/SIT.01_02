Response	headers:
+ The X-Forwarded-Proto - это маркер, который определяет протокол HTTP или HTTPS. 
+ Accept-Encoding - это маркер, который определяет формат сжатия для ответа. 
+ Accept-Ranges - это маркер, который использует сервер, чтобы уведомить клиента о поддержке "запросов по кускам".
+ Strict-Transport-Security - заголовок ответа, который говорит браузеру о том, что сайт имеет доступ только через HTTPS протокол. 
+ ETag -  является идентификатором специфической версии ресурса. Он позволяет более эффективно использовать кеш и сохраняет пропускную способность, позволяя серверу отправлять не весь ответ, если содержимое не изменилось. 
+ Age - Заголовок содержит время в секундах, когда объект был в прокси - кэше.
+ Vary - короче, чтоб узнать нужно из кэша данные доставать или запрос делать. Ну я так поняла. 
+ Expires - заголовок, который содержит дату/время, по истечении которого ответ от сервера считается устаревшим. 
+ X-XSS-Protection - заголовок, являющийся особенностью Internet Explorer, Chrome и Safari, который останавливает загрузку страницы при ри обнаружении (XSS (en-US)) атаки. 
+ Alt-Svc - че-то там про загрузку с других серверов. Указывает что что-то конкретно будет загружено с другого сервера. 

Request headers:
- Sec-Fetch-Dest - это инициатор исходного запроса на выборку, где (и как) будут использоваться полученные. 
- User-Agent - короче, информация о пользователе. 
- Accept - указывает, какие типы контента, выраженные как MIME типы. MIME тип как расширение файла. 
- Authorization -  заголовок, который включает в себя данные пользователя для проверки подлинности пользовательского агента с сервером обычно после того, как сервер ответил со статусом 401 Unauthorized. 
- Referer - Заголовок запроса содержит URL исходной страницы, с которой был осуществлён переход на текущую страницу.
- Origin - тоже что и Referer, только показывает откуда производится загрузка, не показывая весь путь. 
- Content-Type - определяет MIME тип ресурса. 
- Upgrade-Insecure-Requests - заголовок HTTP запроса отправляет сигнал на сервер, выражающий предпочтение клиента в отношении зашифрованного и аутентифицированного ответа, и что он может успешно обработать директиву CSP.

- Cache-Control - общий заголовок используется для задания инструкций кеширования как для запросов, так и для ответов. 
- Cookie - содержит печеньки, раннее отправленные с сервера
